H Luau Style Guide

A strict Roblox Luau coding standard focused on consistency, readability, type safety, and maintainability.

Read the Guide

Core Principles

Strict Typing

All scripts should begin with: 

--!strict

Type annotations are encouraged everywhere possible.

Semicolons

Every statement should end with a semicolon. 

print("Hello");
workspace.Gravity = 196.2;

Functional Programming First

Prefer pure functions and composition whenever practical. Avoid unnecessary mutable state.

Naming Conventions

Element Convention Example
Services PascalCase InventoryService
Variables camelCase playerData
Private Functions camelCase calculateDamage()
Public Functions PascalCase DealDamage()
Classes (OOP) PascalCase PlayerProfile
Data Tables snake_case player_inventory

Variable Declaration

Use const

H Luau Style prefers the newer const keyword for values that should not be reassigned. 

const Players = game:GetService("Players");
const ReplicatedStorage = game:GetService("ReplicatedStorage");

Note: Some Luau environments and tooling may still commonly use local. Teams adopting this guide should ensure their tooling supports the chosen conventions.

Example Service

--!strict

const CoinService = {};
CoinService.__index = CoinService;

type PlayerData = {
    coins: number;
};

const player_data: {[Player]: PlayerData} = {};

function calculateReward(player: Player): number
    return 10;
end;

function CoinService.GiveCoins(
    player: Player,
    amount: number
): ()
    const data = player_data[player];

    if not data then
        return;
    end;

    data.coins += amount;
end;

function CoinService.GetCoins(
    player: Player
): number
    const data = player_data[player];

    if not data then
        return 0;
    end;

    return data.coins;
end;

return CoinService;

Philosophy

"Consistency beats preference. A shared style allows teams to move faster, review code easier, and reduce unnecessary debates."

H Luau Style prioritizes:

  • Predictable naming.
  • Strict typing.
  • Functional thinking.
  • Readable APIs.
  • Maintainable Roblox projects.
